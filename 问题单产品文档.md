# 问题单系统产品文档

## 一、产品概述

### 1.1 产品名称
问题单管理系统（Issue Tracking System）

### 1.2 产品目标
用于个人问题记录、流转与追踪，确保问题可控、可回溯、可闭环。

### 1.3 核心特点
- 标准化问题流转（支持回退）
- 模块化问题分类
- 可搜索、可筛选
- 全流程状态追踪

---

## 二、核心功能设计

注：以下所说的可输入图片输入框，需要可粘贴显示图片，是直接显示图片，不是显示URL地址
### 2.1 问题单创建
- 输入简要描述、问题单详细描述、模块(可选项)自动生成问题单号，用作唯一标识，格式为PRYYYYMMDDHHMMSS，例如：PR20260417223510
- 简要描述只能输入文字。
- 详细描述可输入图片。
- 问题单创建后进入开发实施修改状态
### 2.2 问题单开发实施修改
- 需要输入问题原因，问题修改，自测试，并且选择问题模块后才可以提交。
  问题原因、问题修改、自测试都需要可以输入图片。
- 提交后进入归档状态。
### 2.3 问题单归档
- 输入归档测试描述后，可提交
- 提交后问题单进入关闭状态

### 2.4  模块管理
用户可以自由添加模块，添加后，在问题单创建和开发实施修改，可自由选择已经添加的模块

### 2.3 问题单状态
问题单提交后进入下一状态，回退后回到上一状态。开发实施修改回退回到问题单提交测试阶段，要再次从提交测试阶段提交才能回到开发实施修改
#### 状态机
状态定义：
1. 提交测试（初始）
2. 开发实施修改
3. 归档
4. 关闭
流转规则：
正向：
提交测试 → 开发实施修改 → 归档 → 关闭

逆向：
归档 → 开发实施修改
开发实施修改 → 提交测试 
提交测试 -> 销毁问题单

限制：
只能逐级回退，不允许跨级

### 2.4 问题单列表
- 主页需要显示已有的问题单, 并且需要有过滤选项，过滤问题单状态。
- 列表显示四列：问题单标识，简要描述，当前状态，停留时间（关闭状态不显示）
- 需要有搜索选项，可以搜索问题单号，简单描述，详细描述。
#### 停留时间定义
定义：问题单在当前状态的持续时间
计算方式：
当前时间 - 最后一次状态变更时间

说明：
- 实时计算，不存数据库
- 状态=关闭时不显示

### 2.5 图片存储
图片处理：
方式：本地存储 + 数据库存路径
规则：

- 图片保存到本地目录（如 /images）
- 数据库存储路径
- 支持粘贴上传
- 支持多图

## 四、数据结构设计（核心表）

### 4.1 问题单表（issue）

字段：
- id
- issue_no（唯一标识）
- title（简要描述）
- description（详细描述）
- module_id
- status
- stay_duration
- created_at
- updated_at

### 4.2 模块表（module）

字段：
- id
- name
## 五、开发技术
- 使用PySide6 和 SQLITE。
- 表需要可持久化，软件打开时需要加载历史数据
- 代码至少需要封装成两层UI 和 数据层
