# 问题单管理系统设计文档

## 一、系统架构设计

### 1.1 架构概述

系统采用分层架构设计，分为 **UI层** 和 **数据层** 两层，确保职责分离、易于维护。

```
+------------------+
|      UI层        |  <- 用户界面交互
+------------------+
         |
         v
+------------------+
|    数据层        |  <- 数据存储与访问
+------------------+
```

### 1.2 模块划分

| 层级 | 模块名称 | 职责描述 |
|------|----------|----------|
| UI层 | 主窗口模块 | 问题单列表展示、搜索过滤 |
| UI层 | 问题单创建模块 | 新建问题单表单 |
| UI层 | 问题单详情模块 | 问题单详情查看与编辑 |
| UI层 | 模块管理模块 | 模块的增删改查 |
| 数据层 | 数据库管理模块 | SQLite连接与表初始化 |
| 数据层 | 问题单数据访问模块 | 问题单CRUD操作 |
| 数据层 | 模块数据访问模块 | 模块CRUD操作 |
| 数据层 | 图片存储模块 | 图片本地存储管理 |

## 二、状态机设计

### 2.1 状态定义

| 状态码 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 提交测试 | 问题单初始状态 |
| 1 | 开发实施修改 | 开发人员进行问题修复 |
| 2 | 归档 | 问题修复完成，等待归档测试 |
| 3 | 关闭 | 问题单处理完毕 |

### 2.2 状态流转图

```
                    正向流转（提交）
    ┌──────────────────────────────────────────┐
    │                                          │
    v                                          │
+----------+  提交   +----------+  提交   +----------+  提交   +----------+
| 提交测试  │ -----> | 开发实施  │ -----> |   归档   │ -----> |   关闭   |
|          │ <----- |   修改   │ <----- |          │        |          |
+----------+  回退   +----------+  回退   +----------+        +----------+
     │
     │ 销毁
     v
  [删除问题单]
```

### 2.3 状态流转规则

| 当前状态 | 操作 | 目标状态 | 条件 |
|----------|------|----------|------|
| 提交测试 | 提交 | 开发实施修改 | 必须填写简要描述、详细描述 |
| 开发实施修改 | 提交 | 归档 | 必须填写问题原因、问题修改、自测试，选择模块 |
| 开发实施修改 | 回退 | 提交测试 | 无额外条件 |
| 归档 | 提交 | 关闭 | 必须填写归档测试描述 |
| 归档 | 回退 | 开发实施修改 | 无额外条件 |
| 提交测试 | 销毁 | [删除] | 问题单被删除 |

## 三、数据库设计

### 3.1 问题单表 (issue)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 主键 |
| issue_no | TEXT | NOT NULL UNIQUE | 问题单号 (PRYYYYMMDDHHMMSS) |
| title | TEXT | NOT NULL | 简要描述 |
| description | TEXT | 详细描述 (支持图片路径) |
| module_id | INTEGER | FOREIGN KEY | 关联模块ID |
| status | INTEGER | NOT NULL DEFAULT 0 | 状态码 |
| root_cause | TEXT | 问题原因 (支持图片路径) |
| solution | TEXT | 问题修改 (支持图片路径) |
| self_test | TEXT | 自测试 (支持图片路径) |
| archive_test | TEXT | 归档测试描述 |
| created_at | TEXT | NOT NULL | 创建时间 |
| updated_at | TEXT | NOT NULL | 更新时间 |
| status_changed_at | TEXT | 最后一次状态变更时间 |

### 3.2 模块表 (module)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 主键 |
| name | TEXT | NOT NULL UNIQUE | 模块名称 |
| created_at | TEXT | NOT NULL | 创建时间 |

### 3.3 图片表 (image)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 主键 |
| issue_id | INTEGER | FOREIGN KEY | 关联问题单ID |
| field_name | TEXT | NOT NULL | 所属字段 (description/root_cause/solution/self_test/archive_test) |
| file_path | TEXT | NOT NULL | 图片存储路径 |
| created_at | TEXT | NOT NULL | 创建时间 |

### 3.4 ER关系图

```
+-------------+       +-------------+       +-------------+
|   module    |       |    issue    |       |    image    |
+-------------+       +-------------+       +-------------+
| id          |<------| module_id   |       | id          |
| name        |       | id          |------>| issue_id    |
| created_at  |       | issue_no    |       | field_name  |
+-------------+       | title       |       | file_path   |
                      | description |       | created_at  |
                      | status      |       +-------------+
                      | ...         |
                      +-------------+
```

## 五、数据模型设计

### 5.1 问题单模型 (issue_model.py)

```python
from dataclasses import dataclass
from datetime import datetime
from enum import IntEnum

class IssueStatus(IntEnum):
    """问题单状态枚举"""
    SUBMIT_TEST = 0       # 提交测试
    DEVELOPING = 1        # 开发实施修改
    ARCHIVED = 2          # 归档
    CLOSED = 3            # 关闭

@dataclass
class Issue:
    """问题单数据模型"""
    id: int = None
    issue_no: str = ""
    title: str = ""
    description: str = ""
    module_id: int = None
    status: IssueStatus = IssueStatus.SUBMIT_TEST
    root_cause: str = ""
    solution: str = ""
    self_test: str = ""
    archive_test: str = ""
    created_at: datetime = None
    updated_at: datetime = None
    status_changed_at: datetime = None
    module_name: str = ""  # 关联查询时填充
```

## 七、关键业务流程

### 7.1 创建问题单流程

```
开始
  │
  v
用户点击"新建问题单"
  │
  v
输入简要描述、详细描述，选择模块(可选)
  │
  v
系统生成问题单号 PRYYYYMMDDHHMMSS
  │
  v
保存问题单到数据库 (状态=提交测试)
  │
  v
返回主窗口，列表刷新
  │
  v
结束
```

### 7.2 状态流转流程

```
提交测试状态:
  │
  +-- 提交 --> 开发实施修改状态
  │
  +-- 销毁 --> 删除问题单

开发实施修改状态:
  │
  +-- 提交 (填写原因+修改+自测试+模块) --> 归档状态
  │
  +-- 回退 --> 提交测试状态

归档状态:
  │
  +-- 提交 (填写归档测试) --> 关闭状态
  │
  +-- 回退 --> 开发实施修改状态

关闭状态:
  │
  +-- 流程结束，不可操作
```

### 7.3 图片处理流程

```
用户粘贴图片
  │
  v
富文本编辑器捕获粘贴事件
  │
  v
提取图片数据(支持base64/文件)
  │
  v
生成唯一文件名: {issue_no}/{timestamp}_{random}.png
  │
  v
保存图片到 images/ 目录
  │
  v
将本地路径插入编辑器
  │
  v
显示图片预览
```

## 九、技术选型说明

| 技术 | 版本 | 用途 |
|------|------|------|
| Python | 3.8+ | 开发语言 |
| PySide6 | 6.x | GUI框架 |
| SQLite | 3.x | 嵌入式数据库 |

### 9.1 选择理由

1. **PySide6**: Qt官方Python绑定，支持LGPL协议，适合商业项目
2. **SQLite**: 轻量级嵌入式数据库，无需额外服务，适合桌面应用
3. **分层架构**: UI与数据分离，便于测试和维护
