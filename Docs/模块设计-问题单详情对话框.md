# 问题单详情对话框模块设计文档

## 一、模块概述

### 1.1 模块名称
`issue_detail_dialog.py`

### 1.2 所属层级
UI层

### 1.3 职责描述
展示问题单详情，根据状态提供不同的操作界面，支持提交和回退操作。

## 三、界面元素说明

| 状态 | 可见元素 | 可编辑元素 | 操作按钮 |
|------|----------|------------|----------|
| 提交测试 | 基本信息、简要描述、详细描述 | 无 | 销毁、提交到开发 |
| 开发实施修改 | 基本信息 | 问题原因、问题修改、自测试、模块 | 回退、提交到归档 |
| 归档 | 基本信息、开发信息 | 归档测试描述 | 回退、提交关闭 |
| 关闭 | 所有信息（只读） | 无 | 关闭窗口 |

## 五、状态流转验证规则

### 5.1 提交测试 → 开发实施修改

无需额外验证，直接流转。

### 5.2 开发实施修改 → 归档

| 必填字段 | 说明 |
|----------|------|
| 问题原因 | 不能为空，支持图片 |
| 问题修改 | 不能为空，支持图片 |
| 自测试 | 不能为空，支持图片 |
| 模块选择 | 必须选择一个模块 |

### 5.3 归档 → 关闭

| 必填字段 | 说明 |
|----------|------|
| 归档测试描述 | 不能为空，支持图片 |

## 七、提交/回退流程

### 7.1 提交流程

```
用户点击提交按钮
    │
    v
验证当前状态必填字段
    │
    ├─ 无效 → 显示错误 → 返回
    │
    v 有效
收集表单数据
    │
    v
处理图片数据
    │
    v
更新问题单数据
    │
    v
更新状态到下一状态
    │
    v
更新状态变更时间
    │
    v
保存到数据库
    │
    ├─ 成功 → 发送信号 → 关闭对话框
    │
    └─ 失败 → 显示错误 → 返回
```

### 7.2 回退流程

```
用户点击回退按钮
    │
    v
弹出确认对话框
    │
    ├─ 取消 → 返回
    │
    v 确认
更新状态到上一状态
    │
    v
更新状态变更时间
    │
    v
保存到数据库
    │
    v
发送更新信号 → 关闭对话框
```

## 八、使用示例

```python
from src.ui.issue_detail_dialog import IssueDetailDialog
from PySide6.QtWidgets import QApplication

app = QApplication()

# 打开问题单详情
dialog = IssueDetailDialog(issue_id=1)

# 监听信号
dialog.issue_updated.connect(lambda id: print(f"更新成功: {id}"))
dialog.issue_deleted.connect(lambda id: print(f"删除成功: {id}"))

dialog.exec()
```
