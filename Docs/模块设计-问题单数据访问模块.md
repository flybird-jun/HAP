## 三、SQL语句设计

### 3.1 插入问题单

```sql
INSERT INTO issue (
    issue_no, title, description, module_id, status,
    root_cause, solution, self_test, archive_test,
    created_at, updated_at, status_changed_at
) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
```

### 3.2 查询问题单（带模块名）

```sql
SELECT
    i.*, m.name as module_name
FROM issue i
LEFT JOIN module m ON i.module_id = m.id
WHERE i.id = ?
```

### 3.3 获取所有问题单（带模块名）

```sql
SELECT
    i.*, m.name as module_name
FROM issue i
LEFT JOIN module m ON i.module_id = m.id
ORDER BY i.created_at DESC
```

### 3.4 按状态过滤

```sql
SELECT
    i.*, m.name as module_name
FROM issue i
LEFT JOIN module m ON i.module_id = m.id
WHERE i.status = ?
ORDER BY i.created_at DESC
```

### 3.5 搜索问题单

```sql
SELECT
    i.*, m.name as module_name
FROM issue i
LEFT JOIN module m ON i.module_id = m.id
WHERE
    i.issue_no LIKE ? OR
    i.title LIKE ? OR
    i.description LIKE ?
ORDER BY i.created_at DESC
```

### 3.6 统计各状态数量

```sql
SELECT status, COUNT(*) as count
FROM issue
GROUP BY status
```

## 四、设计要点

### 4.1 依赖注入
通过构造函数注入 DatabaseManager，便于测试和解耦。

### 4.2 数据转换
- 从数据库读取时，将元组转换为 Issue 模型
- 写入数据库时，将 Issue 模型转换为元组
- 时间字符串与 datetime 对象的互相转换

### 4.3 事务处理
涉及多表操作时使用事务保证数据一致性。

### 4.4 空值处理
正确处理 None 值和数据库 NULL。

## 五、使用示例

```python
from src.data.database_manager import DatabaseManager
from src.data.issue_dao import IssueDAO
from src.models.issue_model import Issue, IssueStatus

# 初始化
db = DatabaseManager()
issue_dao = IssueDAO(db)

# 创建问题单
issue = Issue(
    issue_no="PR20260417223510",
    title="登录失败",
    description="点击登录按钮后无响应",
    status=IssueStatus.SUBMIT_TEST,
    created_at=datetime.now(),
    updated_at=datetime.now(),
    status_changed_at=datetime.now()
)
issue_id = issue_dao.create(issue)

# 查询问题单
issue = issue_dao.get_by_id(issue_id)

# 按状态查询
developing_issues = issue_dao.get_by_status(IssueStatus.DEVELOPING)

# 搜索问题单
results = issue_dao.search("登录")

# 更新状态
issue_dao.update_status(issue_id, IssueStatus.DEVELOPING)

# 删除问题单
issue_dao.delete(issue_id)
```
